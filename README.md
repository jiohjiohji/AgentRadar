# AgentRadar

[![Validate](https://github.com/jiohjiohji/AgentRadar/actions/workflows/validate.yml/badge.svg)](https://github.com/jiohjiohji/AgentRadar/actions/workflows/validate.yml)

**AgentRadar reads your project and tells you what tools will help — without you having to search.**

If you're already in Claude Code, just add the plugin. No separate install.

```
/radar scan          # reads your project (or PRD), infers your style, finds gaps
/radar suggest "browser testing"   # describe a need, get compatible matches
/radar check         # flags stale or archived tools and suggests replacements
/radar setup [id]    # installs and configures a tool without leaving your session
```

Or use the standalone CLI:

```bash
agentRadar check     # run in CI — exit 0 healthy, exit 1 action needed
agentRadar scan      # gap detection for a project
agentRadar suggest "browser testing"
```

> **Phase 1 complete.** The `/radar` plugin, CLI, web UI, and API are built. Style-adaptive ranking, intent extraction from markdown, and developer style inference are live. Dataset: 50 tool profiles with curated scores across 6 dimensions. 69 tests passing.

## Why

Every week there are new MCP servers, Claude Code plugins, and orchestration frameworks. Most tool lists are curated by vendors. Most comparisons are vibes-based. Developers waste hours finding, evaluating, and configuring tools that may be stale, incompatible with their stack, or worse than what they already have.

AgentRadar solves this with a curated dataset of tool profiles scored across 6 dimensions — and a plugin that reads your project context, infers your development style, and recommends tools that fit how you work. Vibe coders get low-friction tools. Agent architects get composable tools. Not a leaderboard — a matchmaker.

## Quick start — browse the dataset

**Find a tool by category:**

```
data/tools/          ← 50 tool profiles (YAML)
data/versus/         ← head-to-head comparisons
```

**Read a score:**

Each tool has a `scores` block with six curated dimensions:

| Key | Dimension | What it measures |
|-----|-----------|-----------------|
| `p` | Productivity | Time saved on real tasks |
| `q` | Quality | Output accuracy vs baseline |
| `c` | Cost Efficiency | Token cost relative to benefit |
| `r` | Reliability | Consistency across 5+ runs |
| `x` | Composability | Ease of integrating with other tools |
| `f` | Setup Friction | Time from install to first working result |

Scores are `null` until 2 independent reports exist. `score_confidence` (low/medium/high) is always shown — a score without confidence is meaningless.

## Repository layout

```
data/
  schema.yaml          ← 12-field tool profile schema
  tools/               ← 50 tool profile YAMLs
  tools-index.json     ← auto-generated compact index (build_index.py)
  versus/              ← head-to-head comparison pages
claude-plugin/
  commands/radar.md    ← /radar slash command (scan/suggest/check/setup/show)
api/
  src/                 ← Cloudflare Worker (tools/match/versus/pro routes)
cli/
  src/                 ← TypeScript CLI (check/scan/suggest/watch)
web/
  dist/                ← static site (generated by build_site.py, gitignored)
scripts/
  validate_yaml.py     ← schema validator (run after editing tools/)
  validate_versus.py
  build_index.py       ← generates tools-index.json
  build_site.py        ← generates static HTML for GitHub Pages
  weekly_processor.py  ← stale detection + digest draft + Buttondown send
  crawl.py             ← daily GitHub discovery crawler
templates/
  agentRadar-check.yml ← drop-in GitHub Actions CI health check
.github/
  ISSUE_TEMPLATE/      ← tool-submission.yml
  workflows/           ← validate.yml, daily-crawler.yml, build-site.yml, deploy-worker.yml, weekly-processor.yml
```
